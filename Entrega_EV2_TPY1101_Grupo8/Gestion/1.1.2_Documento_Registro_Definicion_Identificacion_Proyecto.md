# Documento de Registro y Definicion

## Nombre del proyecto
Campus Emprende

## Problema
Los servicios estudiantiles se realizan sin registro formal ni evidencia verificable.

## Oportunidad
Crear una plataforma interna con identidad validada, moderacion y trazabilidad.

## Objetivo general
Transformar actividad estudiantil informal en experiencia profesional verificable.

## Objetivos especificos
- Verificar usuarios.
- Registrar servicios y solicitudes.
- Permitir reviews, comentarios y reportes.
- Organizar evidencia tecnica y de gestion para EV2.

## Alcance
MVP web con backend Spring y frontend React.

## Fuera de alcance
- Pagos.
- Relacion laboral.
- Produccion completa.

## Interesados
- Estudiantes proveedores
- Estudiantes clientes
- Administradores
- Profesor de asignatura

## Restricciones tecnicas reales
- Stack definido Java/React/PostgreSQL.
- Seguridad basada en JWT/OAuth2.
- Dependencia de conectividad y correo.
- Cloud productivo solo referencial.

## Supuestos
- El equipo dispone de acceso al repositorio GitHub.
- La base de datos puede montarse localmente para demo.

## Riesgos
- Evidencia visual pendiente.
- Modulos parciales en EV2.
- Deuda de frontend heredada.

## Criterios de exito
- Estado funcional real documentado.
- Validaciones ejecutadas.
- Entrega organizada en Documentacion, Producto y Gestion.
