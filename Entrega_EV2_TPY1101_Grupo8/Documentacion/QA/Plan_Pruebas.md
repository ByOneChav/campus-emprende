# Plan de Pruebas

## Objetivo
Verificar modulos funcionales del MVP y documentar estados parciales de EV2.

## Alcance
- Auth
- Perfil
- Servicios
- Solicitudes
- Reviews
- Comentarios
- Reportes
- Admin
- Backup y restore

## Tipos de prueba
- Funcionales manuales
- Validacion de requests
- Pruebas unitarias y de slice HTTP
- Validacion de build frontend

## Criterios
- Modulos funcionales deben completar flujo principal.
- Modulos parciales deben quedar documentados con evidencia real o estado honesto.
