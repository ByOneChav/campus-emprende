# Control de Capturas

Las capturas deben incorporarse antes de exportar la entrega final. Este archivo controla la evidencia visual pendiente sin declarar imagenes inexistentes.

- Las imagenes deben obtenerse desde la aplicacion en ejecucion.
- No deben mostrar secretos, tokens completos ni credenciales visibles.
- Si se utiliza un correo real en la demostracion, el dato debe quedar enmascarado en la imagen final.
- El nombre de cada archivo debe seguir un orden correlativo por modulo para facilitar la revision.
- Cuando no exista imagen real, el estado debe mantenerse como `Requiere captura`, `No disponible` o `Evidencia tecnica documental`, segun corresponda.
- Ningun registro puede pasar a `Capturada` si el archivo aun no existe en la ruta declarada.
