# README Capturas
- Las capturas deben ser reales y tomarse desde la aplicacion funcionando.
- No deben incluir secretos, tokens completos ni credenciales visibles.
- Si se usan correos reales, deben enmascararse antes de exportar la entrega.
- Nombrar capturas con prefijo ordenado, por ejemplo `01_registro.png`, `02_login.png`.
- Si una funcionalidad no esta disponible, usar estado `No disponible`, `Parcial` o `Reemplazada por evidencia de codigo`.
- No marcar una captura como `Capturada` si el archivo no existe realmente.
